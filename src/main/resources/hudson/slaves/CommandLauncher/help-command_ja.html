エージェントプログラムの起動に使用するコマンドです。 エージェントプログラムは、エージェントになるコンピュータの制御とマスターとの通信を行います。

この項目にコマンドが指定されたとき、マスターでこのコマンドが実行されます。そして、 Jenkinsは、その実行コマンドが適切なエージェントマシーン上でagent.jarを起動することを想定しています。

agent.jarのコピーはここからダウンロードできます。

シンプルなケースでは、このコマンドは"ssh hostname java -jar ~/bin/agent.jar"のようなものになります。 PATHのようなノード独自の環境変数を設定したり、javaやagent.jarの場所を制御したりできるように、 エージェントに以下のような小さいシェルスクリプトを設置することもできます。 

#!/bin/sh
exec java -jar ~/bin/agent.jar

RSHのようなエージェントマシンでプロセスを起動できるようなコマンドを使用することができます。 ただし、マスターで起動したプロセスの標準入出力が、エージェントマシン上で実行された "java -jar ~/bin/agent.jar"の標準入出力に接続されている必要があります。

エージェントを多数使用する環境の場合、NFSでマウントされた共通の場所から、 agent.jarをロードするなどの構成も検討してください。 そうすれば、Jenkinsを更新するごとに各スレーブのこのファイルを更新する必要がありません。

"ssh -v ホスト名"とすると、接続に関する問題を調べるのに便利です。
